# 📥 ~/Downloads/Transfers — receiving files on `@HOST@`

Incoming transfers land here, via **LocalSend** (phone ↔ PC) and **rsync over
SSH** (PC ↔ PC). On your home network it just works; on untrusted networks you
open a hole *temporarily* from the niri power menu (**Mod+Escape**) and it
auto-closes after 30 minutes.

---

## 📱 LocalSend (phone / tablet ↔ this PC)
- **Home network:** nothing to do — devices detect `@HOST@` automatically
  (`53317/tcp+udp` is open on this machine's firewall zone).
- **Any other network (conference, hotel, café):**
  - Open → power menu **📤 LocalSend share (temp)**  ·  or `localsend-here on`
  - Close → power menu **🔒 LocalSend close**  ·  or `localsend-here off`
  - Auto-closes after 30 min and on reboot — never left open.

## 💻 rsync over SSH (another computer ↔ this PC)
- **This PC → remote** (you push *out*): nothing to set up —
  `rsync -av ~/stuff/ user@remote:/dest/`.
- **Remote → this PC** (something connects *in*): inbound SSH is **key-only** and
  each key is locked to this folder. Set up once:

  1. Authorize the sender's key:
     `add-rsync-key 'ssh-ed25519 AAAA... you@laptop'`
     → that key may ONLY rsync into this folder, upload-only, no deletes, no shell.
  2. Each session: power menu **🔑 SSH/rsync in (temp)** · or `ssh-here on` (30 min).
  3. Sender runs:  `rsync -av ./files/ @HOST@:`   (dest is relative to THIS folder;
     prefix `youruser@` if your name differs, or use `@IP@` if `@HOST@` won't resolve)
  4. Power menu **🔒 SSH/rsync close** · or `ssh-here off` when done.

## 🔒 Why it's safe
- Inbound SSH is keys only (no passwords/root) — `/etc/ssh/sshd_config.d/10-hardening.conf`.
- `sshd` is off by default; `ssh-here` starts it only while needed.
- Transfer keys carry `restrict,command="/usr/bin/rrsync -wo -no-del …"` — nothing but file drops.
- Every temporary opening is runtime-only with a 30-min timeout — wiped on reboot.

## ⚙️ Reproduce on another machine
From your dotfiles:  `just file-transfer`
(then `just file-transfer-harden` on machines that will receive rsync).
